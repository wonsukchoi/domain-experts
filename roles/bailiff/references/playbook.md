# Bailiff — Playbook

## Restraint-decision matrix (filled example)

| Factor | Present? | Points | Notes |
|---|---|---|---|
| Documented escape attempt (any custody setting) | Yes | 2 | County transport, 2013 |
| Violence in custody within past 12 months | No | 0 | No incident reports on file |
| Active gang/security-threat-group affiliation tied to a co-party | Yes | 2 | Shared affiliation with co-defendant's separate case |
| Sentence exposure at or above mandatory-minimum life-adjacent | Yes | 1 | Lead count carries mandatory minimum |
| Prior in-courtroom disruption (this case or prior case) | No | 0 | None on file |
| **Total** | | **5** | |

Cutoffs (illustrative, confirm against local court/sheriff policy): 0-1 = no restraint request needed; 2-4 = concealed restraint (leg brace, under-table cuff), filed as a short written request to the judge; 5+ = concealed restraint plus additional courtroom officer, still filed as a written request — visible restraint is never the default outcome of this table. A visible-restraint outcome requires a separate, specific judicial finding tied to courtroom safety (not escape risk alone), consistent with *Deck v. Missouri*.

**What this table does not decide:** whether to add courtroom officers for a codefendant or spectator threat unrelated to this defendant's custody status — that's a separate line item in the security plan, not folded into the restraint score.

## Screening checkpoint throughput worksheet

```
Expected volume = gallery capacity used + credentialed press + family/support attendees
Per-lane capacity (30-min pre-session window) = (60 / seconds-per-person) × 0.5
Lanes needed = ceiling(expected volume / per-lane capacity)
```

Filled example:
- Expected volume: 45 public + 15 press + 10 family = 70
- Per-person processing time: 25 seconds (walk-through plus bag check) [stated heuristic within the 20-30 second range — confirm against local checkpoint equipment]
- Per-lane capacity in 30 minutes: (60/25) × 30 = 72
- Lanes needed: ceiling(70/72) = 1

Second filled example, higher-volume day (verdict/sentencing):
- Expected volume: 120 public + 30 press + 15 family = 165
- Per-lane capacity: 72 (same processing time)
- Lanes needed: ceiling(165/72) = 3 (round up from 2.29)

**Rule of thumb:** re-run the worksheet for every day with an expected attendance swing of 25% or more from baseline (opening statements, verdict, sentencing) — a lane count set once at trial start and never revisited is the most common screening-capacity failure.

## Sequestration logistics checklist (filled example)

| Item | Detail | Owner |
|---|---|---|
| Lodging | Block-booked hotel, no public booking record under jurors' names | Bailiff + court administrator |
| Transport | Sheriff's van, single manifest, same route/schedule variance daily | Bailiff |
| Meals | Pre-ordered, no menus/media left in common areas | Bailiff |
| Media access | Televisions/devices removed or supervised; daily log of any requested exception | Bailiff |
| Outside contact | Phone calls supervised, logged (caller, duration, no case discussion confirmed) | Bailiff |
| Daily roll call | Headcount at every transition (hotel→van, van→courthouse, courthouse→van, van→hotel) | Bailiff |

A sequestration log missing a headcount at any transition point is treated as an incomplete custody record, not a formality — it's the first document pulled if a juror later reports unsupervised contact.

## Jury note handling — verbatim log format

```
Note #: 3
Time received from jury room: 14:12
Note text (verbatim): "Can we see Exhibit 14 again?"
Time delivered to judge: 14:13
Judge's ruling/response (verbatim): "Yes — bailiff, provide Exhibit 14 to the jury room."
Time response delivered to jury: 14:15
Bailiff signature: [initials]
```

Every note gets its own numbered entry regardless of how minor it appears ("what time do we break") — an incomplete note log is functionally the same evidentiary gap as a missing exhibit custody entry.
