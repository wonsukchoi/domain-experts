# Playbook

Filled templates and reference tables for running sessions, tracking data, and escalating correctly. Defaults, not laws — the supervising SLP's specific plan of care always overrides a generic default here.

## 1. Trial-by-trial data-collection format (filled example)

Session 10, target: /r/ word-initial, plan calls for independent-level probe today.

| Trial # | Target word | Cue level | Response | Note |
|---|---|---|---|---|
| 1 | red | Independent | + | |
| 2 | rabbit | Independent | + | |
| 3 | rain | Independent | − | self-corrected to /w/ |
| 4 | rope | Independent | + | |
| 5 | run | Independent | + | |
| ... | ... | ... | ... | |
| 20 | rock | Independent | + | |

**Session totals: 18/20 correct = 90%, independent cue level.** Never collapse this table into a bare percentage before it reaches the SLP — the per-trial log is the record that survives an audit or a re-review; the percentage is a summary of it, not a replacement.

## 2. Cueing hierarchy (shared vocabulary, low to high support)

| Level | What it looks like | Counts toward "independent mastery" line? |
|---|---|---|
| Independent | No cue given; spontaneous production | Yes |
| Minimal | Brief verbal cue ("remember your tongue tip") | No |
| Moderate | Visual + verbal cue (mirror + verbal reminder) | No |
| Maximal | Full imitative model, or hand-over-hand | No |

Only independent-level trials count toward a documented "independent-level mastery" criterion. Reporting an overall session percentage that blends cue levels ("13/20 = 65% today") without breaking out cue level is the single most common way data becomes uninterpretable to the SLP.

## 3. Common mastery/advancement criteria (menu the SLP selects from — assistant executes, never selects)

| Criterion pattern | Typical use case |
|---|---|
| ≥80% accuracy, independent cue, 3 consecutive sessions | Articulation/phonology drill targets |
| ≥90% accuracy, independent cue, 2 consecutive sessions | Shorter-duration or high-frequency goals |
| ≥80% accuracy in structured drill AND ≥50% in a generalization probe | Goals with an explicit carryover requirement |

The assistant's role stops at reporting which cell of this table the data currently satisfies — never at deciding the client has "graduated."

## 4. Generalization probe schedule and interpretation

| Drill accuracy (independent) | Probe accuracy (conversational) | Gap | What it usually flags for the SLP |
|---|---|---|---|
| 90% | 45% | 45 pts | Skill is acquired in drill, not yet generalized — candidate for a carryover-specific goal |
| 90% | 80% | 10 pts | Generalization likely happening on its own; supports advancing the drill target |
| 60% | 20% | 40 pts | Skill isn't acquired yet at all; drill target itself may need adjustment (SLP call) |

Run the probe on the schedule the plan specifies (commonly every 2–4 sessions once drill accuracy first crosses 80%); report the gap, don't interpret it.

## 5. Weekly data-summary template (fill-in, sent to supervising SLP before the next scheduled review)

```
Client: [initials/ID]          Week of: [date]
Goal(s) addressed: [target, e.g. /r/ word-initial]
Sessions this week: [n]        Total trials logged: [n]

Session-by-session (target / cue level / accuracy):
  [date] — [target] — [cue level] — [x/y = z%]
  [date] — [target] — [cue level] — [x/y = z%]

Criterion status vs. plan: [met / not yet met / partially met — state which specific
  condition of the plan's criterion is or isn't satisfied, do not state "ready to advance"]

Generalization probe (if run): [x/y = z%], gap vs. drill: [n pts]

Anomalies / escalations this week: [none, or: date + description]

Prepared by: [SLPA name, credential]     Reviewed/co-signed by: [SLP name] on [date]
```

## 6. Escalation script — matching urgency to channel

| Situation | Channel | Example wording |
|---|---|---|
| Safety event (choking, fall, injury) | Immediate, in person or phone — do not wait for written note | "Stopping the session — [client] had [event] at [time]; need you now." |
| Unexpected medical symptom (new cough during oral-motor work, sudden fatigue) | Same-day phone or message | "Flagging before next session: [client] showed [symptom] today during [task]; did not continue that activity." |
| Data anomaly (regression, refusal, criterion technicality like the worked example) | Written note attached to next data summary | "Data note for your review before deciding next steps: [specifics]." |
| Caregiver asked a clinical-judgment question | Same-day message, verbatim what was asked and what was said back | "[Caregiver] asked whether to change the goal; I said I'd pass the question to you and gave no opinion — wanted you to know it came up." |

## 7. Caseload-ratio quick reference (examples only — verify current state regulation before relying on a number)

| State (example) | Typical assistant-to-supervising-SLP cap cited in regulation | Note |
|---|---|---|
| Texas | Commonly cited as up to 3 SLPAs per supervising SLP | Verify against current Texas Department of Licensing and Regulation rule text |
| California | Commonly cited as up to 2 SLPAs per supervising SLP | Verify against current California Speech-Language Pathology and Audiology and Hearing Aid Dispensers Board rule text |

Treat any number here as a starting point for the "what's the current cap" question, never as the authoritative figure to schedule against — state rules change and this table is illustrative, not a legal source.
