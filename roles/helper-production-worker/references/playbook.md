# Playbook

## Scope-boundary and observation-relay worked scenario

| Field | Value |
|---|---|
| Assigned task | Stack finished parts on pallet A |
| Observation 1 | Grinding sound, ~8 sec, during one press cycle |
| Observation 2 | 3 of 43 parts show new edge burr defect |
| Helper action on observation 1 | Reported immediately to operator, not filtered as "probably nothing" |
| Helper action on observation 2 | Set defective parts aside separately; did NOT independently scrap or accept (outside scope) |
| Operator action | Checked press, found developing die wear, adjusted |
| Parts caught defective | 3 |
| Parts that could have run before next scheduled quality check | ~50-100 |
| Reduction in defective output from prompt relay | ~95%+ |

**Rule:** relay observations promptly and as-observed; set aside anything ambiguous for the responsible skilled worker's judgment call rather than resolving the quality/adjustment decision independently.

## Support-task priority reallocation guide

| Situation | Priority action |
|---|---|
| Assigned task ongoing, no other need arises | Continue assigned task as instructed |
| A more urgent need appears elsewhere (spill, material shortage at another station, safety hazard) | Reallocate to the more urgent need; inform relevant operator of the switch |
| Unusual observation noticed while performing assigned task | Report promptly without pausing indefinitely — a brief interruption to relay, then resume unless instructed otherwise |
| Multiple operators requesting assistance simultaneously | Triage by safety/urgency first, then communicate expected order to each operator so they can plan around the wait |
| Uncertain whether a task falls in scope | Ask the responsible skilled worker before proceeding — do not guess |

**Rule:** attention reallocation is driven by real-time urgency across the whole work area, not by rigid adherence to whichever task was assigned first.
