# Public Safety Telecommunicator — Vocabulary

### Determinant code
The acuity classification a structured EMD protocol assigns to a call based on the Chief Complaint and Key Questions (e.g. Alpha through Echo in common systems), which determines response level and unit type.
**In use:** "Breathing confirmed absent, that's an Echo — transmit now."
**Common misuse:** treating the determinant code as the complete safety picture, when it only captures medical acuity — a low medical code can still carry a high scene-safety flag (weapon, hazmat) that requires its own response.

### Chief Complaint
The caller's own initial description of the emergency, used to select which protocol card to open — not a diagnosis, just an entry point into the questioning tree.
**In use:** "Chief Complaint was 'unconscious,' so I opened the higher-acuity path pending breathing confirmation."
**Common misuse:** assuming the Chief Complaint alone determines the final determinant code, skipping the Key Questions that actually narrow the tier.

### Key Questions
The specific, ordered follow-up questions a protocol card requires after the Chief Complaint, designed to reach a determinant code as efficiently as possible.
**In use:** "First Key Question on this card is always breathing status."
**Common misuse:** treating Key Questions as optional conversation rather than a required sequence — skipping one to save time is a documented failure pattern in review.

### ANI/ALI
Automatic Number Identification / Automatic Location Identification — the caller's phone number and associated location data displayed to the telecommunicator, sourced differently (and with different accuracy) for landline, wireless, and VoIP calls.
**In use:** "ALI shows a business address, but she's describing a home kitchen — need to confirm."
**Common misuse:** treating ANI/ALI as a GPS-accurate pin for every call type; wireless location carries real uncertainty and often no floor-level data, and VoIP location is only as current as the subscriber's last update.

### Registered Location
The address a VoIP subscriber entered when the service was set up, used as the default location data for that line unless updated — distinct from the caller's actual current location.
**In use:** "Registered Location is three towns over — ask if that's actually where he is."
**Common misuse:** assuming Registered Location auto-updates when a VoIP device moves; it doesn't unless the subscriber manually updates it (the "nomadic VoIP" location problem).

### Dispatch Life Support
The concurrent-processing principle that a unit is dispatched the moment a determinant code is reachable, while the telecommunicator continues working the remainder of the call (background questions, pre-arrival instructions) after transmission rather than before.
**In use:** "Determinant code's in — that's Dispatch Life Support, transmit now and keep working the history questions after."
**Common misuse:** running the full script sequentially and treating dispatch as the last step instead of a mid-call event.

### Pre-arrival instructions
Structured, protocol-driven guidance given to the caller to manage the scene or begin care before responders arrive — e.g. telephone-CPR compression coaching, bleeding control, or scene-safety guidance.
**In use:** "Starting pre-arrival compression instructions now, units are three minutes out."
**Common misuse:** delaying pre-arrival instructions until after all background questions are answered, instead of starting them immediately once dispatch is transmitted.

### Call-processing time
The elapsed interval from call answer to dispatch transmission, tracked against agency or national benchmark targets.
**In use:** "Call-processing time on that one was 38 seconds — inside benchmark."
**Common misuse:** confusing call-processing time (answer-to-dispatch) with total call duration (answer-to-hangup), which includes all the post-dispatch work that shouldn't count against the dispatch-speed benchmark.

### Protocol deviation
Any point where the telecommunicator's actual questioning departs from the card's scripted sequence — whether by necessity (an urgent signal interrupts the script) or by shortcut (skipping a question to save time).
**In use:** "Logged as a protocol deviation — skipped ahead because the caller volunteered the weapon information unprompted."
**Common misuse:** treating "deviation" as inherently bad; a documented, reasoned deviation (escalating early on a clear signal) is expected and defensible, while an undocumented one (skipping a safety question) is not.

### Co-response
A call requiring more than one type of responding unit — e.g. a medical call that also needs law enforcement because of a scene-safety flag, dispatched together rather than sequentially.
**In use:** "Flagging this as co-response — weapon mentioned, need PD en route with the medic unit."
**Common misuse:** assuming the medical determinant code alone drives all dispatch decisions, missing that safety flags can require an additional unit type independent of medical acuity.

### Callback
An attempt to re-contact a caller after a dropped or disconnected call, required by policy before closing a case unless units already on scene confirm no further contact is needed.
**In use:** "Line dropped — attempting callback now before I close this out."
**Common misuse:** treating a dropped call as automatically resolved or lower-priority; a drop can just as easily mean the caller is now unable to speak.
