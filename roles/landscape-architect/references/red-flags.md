# Red flags

### Cut/fill imbalance exceeding 10% of total earthwork volume
- **Usually means:** grading concept was fixed to match the rendering before the volumes were checked; second most likely is a fixed building pad elevation forcing an import/export the client hasn't priced.
- **First question:** "Has anyone run the earthwork takeoff against this grading plan yet?"
- **Data to pull:** cut/fill volume takeoff from the current grading plan revision.

### Planting palette proposed outside the site's mapped hardiness zone
- **Usually means:** palette was selected from a stock catalog or a reference project in a different climate; second is a client-driven species request without a site-suitability check.
- **First question:** "What's the site's actual USDA zone, and does every species on this list survive it?"
- **Data to pull:** USDA hardiness zone lookup for the site coordinates, cross-referenced against the plant schedule.

### Net new impervious area within 20% of the jurisdiction's stormwater review trigger
- **Usually means:** the design is one parking-space addition away from triggering full detention review mid-design; second is the trigger threshold hasn't been confirmed with the local authority.
- **First question:** "Have we confirmed this jurisdiction's exact impervious-area trigger, or are we assuming a typical number?"
- **Data to pull:** local stormwater ordinance impervious-area threshold and current impervious-area takeoff.

### Large-caliper stock specified without an establishment irrigation plan
- **Usually means:** client requested an instant-mature look and the budget conversation about shock mortality never happened; second is a value-engineering pass removed irrigation without revisiting caliper size.
- **First question:** "Who's paying for replacement stock if 15–20% of these die in year one?"
- **Data to pull:** nursery mortality data for the specified caliper/species combination and the irrigation line item in the budget.

### Planting bed or hardscape element crossing a recorded easement
- **Usually means:** the survey's easement layer wasn't cross-checked against the site plan before design; second is a late-added utility easement post-dating the base survey.
- **First question:** "When was this survey's easement data last verified against county records?"
- **Data to pull:** title report or recorded plat showing all easements, overlaid on the current site plan.

### Single-species planting covering more than roughly 30% of a contiguous planted area
- **Usually means:** cost or aesthetic uniformity drove the selection without a pest/disease diversification check; second is a client aesthetic preference overriding the diversification recommendation.
- **First question:** "What happens to this area if one pest or pathogen hits this species?"
- **Data to pull:** regional pest/disease advisories for the proposed species (e.g., state forestry or extension service alerts).

### Detention pond sized to a storm event smaller than the jurisdiction's code minimum
- **Usually means:** the sizing calc used an outdated or wrong design-storm assumption; second is a value-engineering reduction applied after the civil sizing was finalized.
- **First question:** "What design storm does this jurisdiction's code actually require, and does the sizing calc match it?"
- **Data to pull:** local stormwater code design-storm requirement and the detention sizing calculation sheet.

### Grading plan issued for construction without a percolation test on unknown soils
- **Usually means:** the schedule pressured skipping the test in favor of a generic infiltration assumption; second is a prior geotechnical report being assumed still valid for a different area of a large site.
- **First question:** "Where on this site was the infiltration rate actually tested, versus assumed?"
- **Data to pull:** geotechnical/percolation test report and its coverage area relative to the drainage feature being sized.

### Plant schedule quantities that don't reconcile with the planting plan drawing
- **Usually means:** a late plan revision moved beds without updating the schedule; second is a scale/count error in the takeoff.
- **First question:** "Has anyone recounted the plan against this schedule since the last revision?"
- **Data to pull:** current planting plan drawing and plant schedule, counted side by side.

### Client-requested hardscape expansion added after stormwater sizing was finalized
- **Usually means:** scope crept after the civil package was locked, without a re-run of the impervious-area calc; second is a phased-project assumption that the addition wouldn't count toward the site total.
- **First question:** "Has the stormwater sizing been re-run since this hardscape addition was approved?"
- **Data to pull:** revised impervious-area takeoff including the added hardscape.
