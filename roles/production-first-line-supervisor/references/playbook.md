# Playbook

## Shift handoff template

Fill and hand to the incoming supervisor at every shift change; incoming supervisor confirms receipt.

| Section | Content |
|---|---|
| Production status | Actual vs. target (full-shift and pro-rata), with specific attributable causes for any shortfall |
| Open quality issues | Any active holds, out-of-spec trends, or pending investigations — station/line, status, next step |
| Open maintenance issues | Any active tickets, equipment status, expected resolution |
| Open safety issues | Any near-misses, incidents, or hazard reports from this shift |
| Personnel matters | Staffing gaps, performance/coaching items in progress, anything the next shift needs to know |
| Escalations made this shift | What was escalated, to whom, current status |
| Anything else non-obvious | Anything an incoming supervisor would otherwise have to discover independently |

**Rule:** silence on a section means "nothing open," not "not checked" — actively confirm each section rather than skipping ones that seem routine.

## Escalation triage table

| Situation | Escalate? | To whom |
|---|---|---|
| Reject/defect rate spike with unclear cause | Yes — pause affected station, escalate for diagnosis | Maintenance and/or Quality |
| Routine equipment adjustment within normal operating parameters | No — line-level authority | N/A |
| Safety near-miss or hazard observed | Yes — always report | EHS |
| Personnel performance issue, first occurrence, non-safety | No initially — direct corrective coaching at line level | Escalate only if recurring or safety-related |
| Personnel safety-behavior violation | Yes — immediate correction, and report per facility policy | EHS / HR per severity |
| Production behind pace, no quality/safety factor | No — line-level management (adjust staffing, sequencing) | Escalate only if a systemic capacity issue is suspected |
| Any issue requiring technical diagnosis beyond line-level knowledge | Yes | Relevant technical function (maintenance, quality, engineering) |

## Quality-vs-production conflict resolution guide

1. Identify the specific quality/safety signal and its trend (is it worsening, stable, or already resolved?).
2. Estimate the cost of addressing it now (production time lost) versus the cost of continuing without addressing it (escape risk, safety exposure, potential for the issue to worsen).
3. Default to treating quality/safety as the binding constraint unless there's a clear, documented reason this specific signal doesn't carry meaningful risk if unaddressed for the remainder of the shift.
4. Scope any pause/hold as narrowly as possible (the specific station/process step) rather than stopping the entire line, to preserve production continuity where the risk doesn't actually extend there.
5. Document the decision, the reasoning, and the outcome for shift handoff and any post-shift review.
