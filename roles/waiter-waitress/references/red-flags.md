# Red flags

Smell tests a veteran server or floor manager catches in the first few minutes of watching a section run.

### A table has gone >8 minutes since the last server touch during an active course

- **Usually means:** the server is triaging by whoever's speaking loudest, not by the clock — a quiet table is the one most likely to leave a low tip without ever flagging a complaint.
- **First question:** "When did anyone last make eye contact with that table?"
- **Data to pull:** POS timestamp of last item rung or modified for that table versus current time; server's own section map if the house uses one.

### Four or more entrée tickets fire to the same station within a 5-6 minute window

- **Usually means:** tables were seated too close together by the host (a stacking failure) and the server fired on autopilot instead of sequencing the fire — the line will back up and plates will arrive unevenly hot.
- **First question:** "Were these tables sat within the same 10-minute window?"
- **Data to pull:** host stand seating timestamps for the section; kitchen display system (KDS) fire-to-ready times for the affected tickets.

### A server's average tip percentage sits well below the house average despite average or above-average sales

- **Usually means:** service pacing or warmth is the actual gap, not menu mix or price point — sales volume can mask a service problem that a tip-percentage comparison exposes.
- **First question:** "Is this server's covers-per-hour normal, or are they turning tables fast and losing the attentiveness that drives tip percentage?"
- **Data to pull:** tip percentage by server over a full month, not one shift; table-turn time per server; any recent guest complaints tied to that section.

### An allergy is noted only as a free-text ticket comment or spoken to the server and nowhere else

- **Usually means:** the disclosure never reached the line cook who actually plates the dish — free-text notes get missed under rush, and a verbal-only promise leaves no record at all.
- **First question:** "Show me the allergen modifier code on this ticket, not the comment field."
- **Data to pull:** POS ticket detail for the order; house allergen matrix; whether the expo confirmed verbally at fire time.

### The host has seated an entire section within a 5-10 minute window

- **Usually means:** a pacing failure at the host stand, often from over-correcting a long wait list, that guarantees every table in that section will hit the same course stage at the same time.
- **First question:** "What's the seating-time spread across this server's tables right now?"
- **Data to pull:** host stand seating log timestamps; whether this is a one-off or a recurring pattern for this host on this shift type.

### Tip-out is calculated as a flat percentage of sales rather than of tips collected, and nobody has explained the difference to new servers

- **Usually means:** a server who gets tipped poorly on a table still owes the same tip-out, which quietly disincentivizes effort on tables already read as low tippers — and new hires often don't realize this until their first bad night.
- **First question:** "Do you know whether tip-out here comes off your sales or off your tips?"
- **Data to pull:** house tip-out policy documentation; a sample settlement sheet from a recent shift, walked through line by line.

### A server's direct wage plus reported tips falls under the federal (or applicable state) minimum for any pay period

- **Usually means:** either a slow shift genuinely didn't generate enough tips and the employer owes a make-up payment, or tips are being under-reported — both need to be resolved before the next pay cycle, not carried forward.
- **First question:** "Pull this pay period's hours, direct wage, and reported tips — does direct wage plus tips clear $7.25/hr for every hour, or the applicable state minimum if higher?"
- **Data to pull:** payroll register for the period; state tipped-minimum-wage rule (several states require full minimum wage with no tip credit at all); reported tip totals versus POS-recorded tips for the same shifts.

### The same upsell script is used on every table regardless of how the table has responded

- **Usually means:** the server has stopped reading the table and is running a memorized pitch — trust and check average both erode within a single night once a table clocks the script.
- **First question:** "What did this table actually ask for, and did the recommendation change based on that?"
- **Data to pull:** manager floor observation notes; comparison of upsell attach rate on tables that gave a clear signal (price-conscious, celebratory) versus the pitch actually delivered.

### A comp or discount is applied at the table level without a manager override on file

- **Usually means:** a server unilaterally promised a fix to a guest under pressure rather than escalating — which either costs the house money it didn't authorize or, if reversed later, turns one unhappy moment into two.
- **First question:** "Who approved this comp, and was the guest told before or after it was decided?"
- **Data to pull:** POS void/comp report with manager-approval flag; server notes on what was said to the table in the moment.
