# Working vocabulary

Terms of art practitioners use precisely, that generalists misuse. Load this when writing or reviewing content that needs to sound like an actual natural sciences/R&D manager, not a generalist describing one.

### Expected information value
How much a project's result (in either direction) would change a real downstream decision — a portfolio prioritization concept distinct from a project's cost or scientific interest alone.
**In use:** "This project scores low on info value — even a positive result wouldn't change what we do next, so it's not worth the allocation regardless of how interesting it is."
**Common misuse:** Conflated with "how interesting is this scientifically" — a scientifically interesting question whose answer (either direction) wouldn't change any actual decision has low expected information value for portfolio purposes, even though it might be worth pursuing for other reasons (publication, training, capability building).

### Go/no-go criterion
A pre-defined, specific threshold set before data collection that determines whether a program continues, pivots, or stops at a checkpoint.
**In use:** "The go/no-go was set at 25% improvement before we started — 12% doesn't meet it, regardless of how the team feels about the progress."
**Common misuse:** Defined loosely or after the fact ("we'll know it when we see it") rather than as a specific number set in advance — a criterion set after data collection is vulnerable to being adjusted to match whatever result actually came in.

### File-drawer problem / publication bias
The tendency for null or negative results to go unpublished or unreported while positive results are preferentially shared, distorting the overall visible evidence base.
**In use:** "We've published three positive trials on this compound class — how many negative ones are sitting in someone's file drawer that we don't know about?"
**Common misuse:** Assumed to be purely an external, academic-publishing problem — the same dynamic operates inside a single organization's internal reporting if null results aren't given equal visibility.

### Pre-registration
Documenting a hypothesis and analysis plan before data collection begins, to prevent post-hoc rationalization of ambiguous or unexpected results.
**In use:** "This wasn't pre-registered, so before we treat this subgroup finding as real, let's be honest that we're now in exploratory territory, not confirmatory."
**Common misuse:** Skipped as unnecessary overhead for internal (non-publication) research — the value of pre-registration (preventing after-the-fact rationalization) applies just as much to internal go/no-go decisions as to academic publications.

### Replication
Independent reproduction of a result, ideally by a different team/lab under the same or closely similar conditions, as the actual standard for treating a finding as reliable.
**In use:** "One lab, one result — that's a hypothesis. We need at least one independent replication before this drives the funding pivot."
**Common misuse:** Confused with a second run by the same team/lab — same-team replication doesn't rule out shared systematic errors, equipment calibration issues, or unconscious bias the way independent replication does.

### Sunk cost (in R&D decisions)
Resources already spent on a program, which are — by standard decision theory — irrelevant to whether continuing to invest further is currently the best use of resources.
**In use:** "The $1.2M already spent doesn't belong in this decision — the question is whether the next $400K is the best use of that money right now, compared to the alternatives."
**Common misuse:** Treated as a legitimate reason to continue a program ("we've come this far") — sunk cost is specifically the cost that should NOT factor into a forward-looking continue/stop decision, even though it's psychologically one of the strongest pulls in that direction.

### Diminishing returns (in iterative R&D)
A pattern where each successive round of effort/investment produces a smaller improvement than the last, signaling the current approach may be approaching its practical ceiling.
**In use:** "Three iterations, +3%, +2%, +1% — that's a diminishing-returns trend, not a sign we're about to break through."
**Common misuse:** Ignored in favor of extrapolating the average improvement rate forward — a diminishing trend (each step smaller than the last) implies a much lower ceiling than a naive average-rate extrapolation would suggest.

### Scoped pivot
Redirecting a program's accumulated expertise/assets toward a narrower or differently-defined question, rather than either continuing the original program unchanged or abandoning the work entirely.
**In use:** "We're not continuing toward the original 40% target, but the 12% we did achieve might have standalone value for [narrower application] — that's a scoped pivot, not a full stop."
**Common misuse:** Treated as a binary with "continue" or "kill" — a scoped pivot is a distinct third option that preserves accumulated value without pretending the original program met its bar.
