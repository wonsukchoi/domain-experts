# Red flags

### Product target tightened (finer spec) without adjusting classifier cut point, relying on increased mill residence time/power alone
- **Usually means:** the classifier still allows oversize material through regardless of overall grinding intensity, and the target may not actually be met.
- **First question:** does the classifier cut point setting actually match the target's finest requirement?
- **Data to pull:** classifier cut point setting compared against target product size specification.

### Circulating load ratio not monitored as a distinct diagnostic variable, with mill performance evaluated only by power draw and throughput
- **Usually means:** an out-of-optimal-range circulating load, too high or too low, could be reducing grinding efficiency undetected.
- **First question:** what is the current circulating load ratio versus the circuit's documented optimal range?
- **Data to pull:** current circulating load ratio compared against the circuit's documented optimal range.

### Grinding media size/charge reused from a prior job without matching to the current target particle size reduction
- **Usually means:** grinding energy delivered may be mismatched to the actual size reduction task, wasting energy or under-grinding.
- **First question:** does the media size/charge in use match the configuration appropriate for the current target size reduction?
- **Data to pull:** media size/charge in use compared against the media configuration appropriate for the current target size reduction.

### Mill operating speed not verified/calculated as a percentage of critical speed appropriate for the intended grinding action
- **Usually means:** the mill may be operating in an inefficient regime — too fast, causing centrifuging rather than effective cascading motion.
- **First question:** what does actual mill RPM show against calculated critical speed and the target percentage for this grinding action?
- **Data to pull:** actual mill RPM compared against calculated critical speed and the target percentage for this grinding action.

### Product size distribution inferred from mill power draw, throughput, or elapsed time rather than actual sieve/laser analysis
- **Usually means:** feed hardness, moisture, or classifier condition changes could produce an off-spec product undetected by process parameters alone.
- **First question:** what does an actual sieve/laser analysis show against specification for this batch?
- **Data to pull:** actual sieve/laser analysis result compared against specification, for the current batch/run.

### A recurring off-spec product size pattern for a specific circuit/target appears without a traced cause
- **Usually means:** a systemic process issue — classifier cut point, circulating load, media condition, feed characteristics — rather than isolated one-off run variation.
- **First question:** does the off-spec occurrence log correlate with a specific process parameter?
- **Data to pull:** off-spec occurrence log compared against process parameter logs (classifier setting, circulating load, media, feed characteristics) over the same period.

### Feed material characteristics change without a corresponding reassessment of classifier/media/circulating-load settings
- **Usually means:** settings validated for one feed type may not produce the same product size result on a different feed.
- **First question:** do current feed characteristics match what the current settings were originally validated for?
- **Data to pull:** current feed characteristics compared against the feed characteristics the current process settings were originally validated for.
