# Playbook

## Route casing sequence (filled example)

Route 12, morning case, before departure:

| Step | Input | Action | Time (standard) |
|---|---|---|---|
| 1 | DPS trays (3), pre-sequenced letters | Verify tray order matches route sequence, load into satchel/vehicle in order | 10 min |
| 2 | Flats (non-DPS), ~90 pieces | Hand-case into delivery-point order using the case | 35 min |
| 3 | Forwards (12 pieces, active hold/forward orders) | Cross-check each against current forwarding-order system before casing in — do not case in an item whose hold is still active | 8 min |
| 4 | Parcels (22), too large for standard slots | Sequence separately by delivery point, flag any exceeding CBU/mailbox capacity for attempted-delivery handling | 12 min |
| 5 | Route book review | Check for new dog flags, CBU issues, access notes added since last shift | 5 min |

Total casing time: 70 minutes, within the route's evaluated office-time standard of 75 minutes.

## Count and Inspection request (filled example)

Used when actual route time has run over standard for 3+ consecutive weeks and the cause isn't a one-off (holiday volume, weather).

**Data to gather before requesting:**

| Metric | Value |
|---|---|
| Evaluated standard (last Count and Inspection) | 7.5 hrs/day, 450 delivery points |
| Actual average, trailing 4 weeks | 8.9 hrs/day |
| Overage | 1.4 hrs/day |
| Delivery points added since last evaluation | 38 (Phase 2 subdivision) |
| Estimated time added by new points (at 1 min/point standard) | 0.63 hrs/day |
| Unexplained overage | 1.4 − 0.63 = 0.77 hrs/day |
| Unexplained overage, weekly (6-day delivery week) | 0.77 × 6 = 4.62 hrs/week |

**Request language (verbal or written to supervisor):**

> Requesting a Count and Inspection for Route 12. Evaluated standard is 7.5 hours/day; trailing 4-week actual average is 8.9 hours/day, a 1.4-hour/day overage. The Phase 2 subdivision added 38 delivery points this quarter, accounting for an estimated 0.63 hours/day at the route's own per-point standard — leaving 0.77 hours/day (4.6 hours/week) unaccounted for. Requesting formal re-evaluation rather than continuing to absorb the gap.

## CBU exception handling (filled example)

CBU at 4th and Elm serves 14 addresses. Lock reported jammed for the second time this month.

1. Do not force the lock or attempt a workaround delivery (e.g. leaving mail loose) — that creates a security and liability exposure for all 14 addresses.
2. Scan all 14 delivery points as "attempted — access issue," not as delivered.
3. Log the CBU fault in the maintenance ticket system, noting it's the second report this month (escalates from routine repair to priority replacement review).
4. Note the outage in the route book so any substitute carrier covering the route knows not to expect normal CBU access until resolved.
5. Follow up on ticket status before the next scheduled delivery day; if unresolved, repeat the attempted-delivery scan process rather than delivering into a compromised unit.
