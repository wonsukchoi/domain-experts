# Red flags

### Loops or visible loose thread on the underside of a seam
- **Usually means:** bobbin tension is out of balance relative to top tension, not necessarily a top-tension problem.
- **First question:** has bobbin tension been checked directly, or was only the top tension adjusted in response?
- **Data to pull:** current top and bobbin tension settings, a test seam checked on both sides.

### A machine started on a new bundle without checking needle/seam-type/tension settings against the new fabric
- **Usually means:** the machine still carries settings from the prior job, which may not match the current fabric.
- **First question:** does the fabric type/weight for this bundle match the prior job, or has it changed?
- **Data to pull:** current bundle's fabric specification, machine's current needle/seam-type/tension settings.

### A seam tested only flat and unstressed on a stretch fabric
- **Usually means:** the seam's actual functional requirement (stretching with the fabric) hasn't been verified.
- **First question:** has a stretch test been performed, simulating the garment's actual use condition?
- **Data to pull:** fabric's stretch classification, whether a stretch test was performed and its result.

### Stitch density increased to add strength on lightweight or loosely woven fabric
- **Usually means:** the fabric's tear resistance at that density hasn't been checked, risking a weakened seam line from excess perforation.
- **First question:** has the fabric's tear resistance actually been tested at this stitch density, or was density simply increased on the assumption more is stronger?
- **Data to pull:** fabric weight/construction, tear test result at the current stitch density if performed.

### Skipped stitches or fabric damage attributed to tension without checking needle condition
- **Usually means:** a burred, bent, or wrong-type needle may be the actual cause, not a tension imbalance.
- **First question:** has the needle been inspected for damage or confirmed as the correct type for this fabric?
- **Data to pull:** needle inspection/replacement log, needle type/size against the fabric's specification.

### A production bundle proceeding without a start-of-bundle test seam
- **Usually means:** piece-rate time pressure led to skipping the verification step before committing to full quantity.
- **First question:** was a test seam run and checked before production started, or did production begin immediately?
- **Data to pull:** bundle start log/ticket, any test seam record.

### A quality issue discovered partway through a bundle, with unclear scope of affected units
- **Usually means:** the specific units produced before the issue was caught haven't been identified for containment.
- **First question:** how many units were produced before the issue was found, and have they been separately flagged?
- **Data to pull:** production sequence/count at the time the issue was found, units flagged for rework or verification.

### A seam type used that doesn't match the fabric's stretch classification
- **Usually means:** the seam construction (e.g., straight lockstitch) doesn't accommodate the fabric's actual elongation behavior.
- **First question:** does the specified/used seam type match this fabric's stretch classification (woven/non-stretch vs. knit/stretch)?
- **Data to pull:** fabric stretch classification, seam type specification for this bundle.

### Thread breaking repeatedly during a run on the same machine
- **Usually means:** could be tension imbalance, wrong thread weight for the fabric/needle combination, or a needle/machine mechanical issue — needs diagnosis, not just re-threading.
- **First question:** does the breakage correlate with a specific tension setting, thread type, or needle condition?
- **Data to pull:** thread type/weight used, current tension settings, needle condition.

### An operator reporting they "always use" a specific needle/tension setting regardless of fabric
- **Usually means:** settings aren't being verified per job, relying on habit rather than the current fabric's actual requirements.
- **First question:** does this operator's standard setup match what different fabric types in the current production mix actually require?
- **Data to pull:** the range of fabric types this operator handles, whether settings are actually adjusted between them.
