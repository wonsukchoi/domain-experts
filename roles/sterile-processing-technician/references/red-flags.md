# Red flags

Smell tests a senior sterile processing technician catches before a tray leaves the department. Format for each: what it usually means, the first question to ask, and the data to pull.

### Chemical indicator passed but the sterilizer printout shows a parameter deviation

**Usually means:** either a genuine cycle abort/underrun that the CI happened not to register (CIs confirm exposure to a parameter, not the full cycle), or a sensor fault on the printout side. Releasing on the CI alone when the printout disagrees is the single most common way an under-processed load reaches the OR.

**First question:** "What exact temperature, pressure, and time did the printout log for this cycle, and does it match the validated parameters for this load type?"

**Data to pull:** the cycle printout in full, the validated cycle parameters for that sterilizer/load combination, and the maintenance log for that unit's temperature/pressure sensors.

### Wet-pack rate on one sterilizer above ~2% of loads over a week

**Usually means:** loading pattern (trays too dense or stacked), a water-quality issue specific to that unit's feed line, or a drying-phase setting that's been shortened — rarely staff technique alone, which would show up across sterilizers, not concentrated on one.

**First question:** "Is this pattern on one sterilizer or across all of them, and has the load-density or drying-cycle setting changed recently?"

**Data to pull:** wet-pack incident log by sterilizer, loading diagrams for the affected loads, drying-phase cycle parameters and any recent service/firmware history on that unit.

### Implant tray released before BI result is back

**Usually means:** turnover pressure overrode the mandatory hold — a bypass of the one rule that has no CI-based exception. This is the highest-severity finding in the department because it directly reaches a patient with no monitoring safety net beyond physical/chemical.

**First question:** "What was the clinical justification for releasing before BI confirmation, and who authorized it?"

**Data to pull:** the tray tracking record showing release time vs. BI read time, the facility's implant-release policy, and whether the case proceeded or was delayed for a substitute set.

### Biological indicator positive with the paired negative control also showing growth

**Usually means:** a contaminated test lot or improper control handling, not necessarily a sterilizer failure — but it must be ruled out with a fresh lot before either concluding "sterilizer's fine" or launching a full recall on bad information.

**First question:** "Is this an isolated BI lot number, or has any other unit's control also shown growth this week?"

**Data to pull:** BI lot numbers across all sterilizers for the week, manufacturer lot-recall notices, and incubator calibration/temperature logs.

### Same instrument tray shows repeat count discrepancies across multiple assemblies

**Usually means:** a systemic issue — a damaged count sheet, an instrument routinely mis-sorted at decontamination into the wrong tray's bin, or a tray configuration that was changed without updating the count sheet.

**First question:** "Is the count sheet current with the tray's actual configuration, and when was it last updated?"

**Data to pull:** count discrepancy log for that tray ID over the last quarter, the current count sheet revision date, and decon sorting records for the instrument in question.

### A sterilizer's BI testing frequency has lapsed (no negative BI on record in the expected window)

**Usually means:** a scheduling or staffing gap, not a sterilizer problem per se — but every load run during the lapse is undocumented risk that has to be treated the same as a recall-scope question once discovered.

**First question:** "What is the last date we have a confirmed-negative BI on this unit, and how many loads have run since?"

**Data to pull:** BI testing log for the unit, load log for the same period, and tray tracking to determine how many of those loads have already been used.

### Immediate-use steam sterilization (IUSS/flash) used routinely to cover instrument shortages

**Usually means:** inadequate instrument inventory or turnover scheduling is being solved by skipping the drying step on a recurring basis, not by an actual emergency — published data ties routine flash use to elevated surgical-site-infection risk, and "we always run one flash load per day" is the tell that it has become workflow, not exception.

**First question:** "How many IUSS cycles ran this week, and for how many was a substitute wrapped/sterile set genuinely unavailable?"

**Data to pull:** IUSS log with stated clinical justification per use, instrument inventory counts for the sets most frequently flashed, and case scheduling data to see if shortages cluster around specific high-volume days.

### A wrapped package looks intact but has been stored on a bottom shelf near a floor vent or high-traffic aisle

**Usually means:** event-related sterility has been quietly compromised by storage conditions (temperature/humidity cycling, handling frequency) even though the date and visible seal look fine — the failure mode this policy exists to catch is exactly the one that looks fine on cursory inspection.

**First question:** "What's the storage location's humidity and traffic exposure, and has this shelf had a integrity issue before?"

**Data to pull:** storage area environmental logs if available, shelf/location history for damaged-pack reports, and the facility's event-related sterility policy defining acceptable storage conditions.
