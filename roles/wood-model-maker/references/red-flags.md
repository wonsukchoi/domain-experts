# Red flags

### Multiple similar features measured by stepping off from the previously cut feature rather than the original source drawing
- **Usually means:** scale conversion rounding/reading error is compounding progressively across the sequence.
- **First question:** was each feature's position calculated independently from the original full-scale source, or chained from the prior cut?
- **Data to pull:** the measurement method used, original source drawing dimensions for comparison.

### A model dimension treated as "close enough" because the absolute physical error seems tiny
- **Usually means:** the error hasn't been projected to full scale to check its actual significance.
- **First question:** what does this model-scale error represent once multiplied by the scale ratio?
- **Data to pull:** the model-scale error magnitude, the scale ratio, calculated full-scale equivalent.

### A model material selected to literally match the full-scale material without testing how it reads at model scale
- **Usually means:** the material's visual properties (gloss, texture, color saturation) may not translate correctly when scaled down.
- **First question:** has a sample been tested at the model's actual scale and intended viewing distance, or was the material chosen by full-scale logic alone?
- **Data to pull:** material sample test results if performed, the model's intended viewing distance/context.

### A model built without clarifying its actual purpose with the requester beforehand
- **Usually means:** precision effort may be misallocated — over-engineered for a concept model, or under-specified for a functional fit-check model.
- **First question:** has the model's actual purpose (visual communication vs. dimensional verification) been confirmed?
- **Data to pull:** the request/brief for the model, any purpose clarification on record.

### A one-time review model built to the same robust construction standard as a long-term display or shipping model
- **Usually means:** effort/cost may be misallocated relative to the model's actual anticipated handling and lifespan.
- **First question:** what is this specific model's anticipated handling (one-time review, repeated shipping, long-term display)?
- **Data to pull:** the model's intended use/lifespan, construction approach used.

### A functional/fit-check model prioritizing surface finish and appearance over dimensional accuracy
- **Usually means:** the model's actual purpose (functional verification) isn't being served by where precision effort was spent.
- **First question:** does the model's dimensional accuracy actually meet the fit-check requirement, regardless of surface finish quality?
- **Data to pull:** dimensional verification data, the model's stated functional purpose.

### A visual/concept model's proportions drifting noticeably across a repeated feature sequence
- **Usually means:** a chained-measurement compounding error, consistent with not referencing the original source for each feature.
- **First question:** does the drift pattern match a progressive, compounding error signature (increasing across the sequence)?
- **Data to pull:** measured positions of each feature in the sequence, comparison to calculated positions from the source.

### A scale conversion calculation performed without documenting the scale ratio and source dimension used
- **Usually means:** the basis for the model's dimensions isn't traceable, making it hard to verify or reproduce.
- **First question:** does the project record show the scale ratio and source dimensions for each converted feature?
- **Data to pull:** the model's project/build record.

### A model delivered without a final verification check against its actual stated purpose
- **Usually means:** the model may not have been confirmed to actually serve its intended function (visual proportion check or dimensional check) before delivery.
- **First question:** was a purpose-appropriate verification (proportion review or dimensional measurement) performed before delivery?
- **Data to pull:** any verification record, the model's stated purpose.

### A design team misinterpreting a concept model's precision as dimensionally exact, or a functional model's finish as production-representative
- **Usually means:** the model's actual scope/purpose wasn't clearly communicated to the viewing audience.
- **First question:** did the delivery communication clearly state what the model does and doesn't represent accurately?
- **Data to pull:** delivery notes/communication accompanying the model.
