# Tutor — Playbook

## Diagnostic pre-assessment (filled example)

12-item diagnostic for a two-step-equations unit, items tagged to skill cluster before scoring (not after) so the breakdown is available the moment answers come in.

| Item # | Skill cluster | Correct? |
|---|---|---|
| 1, 2 | Integer operations | Y, Y |
| 3 | Integer operations | Y |
| 4, 5 | Distributive property | N, N |
| 6, 7, 8 | Fraction/decimal operations | Y, N, N |
| 9, 10, 11, 12 | Two-step equations (current unit) | N, Y, N, N |

Score by cluster, not just overall: Integer ops 3/3 (100%), Distributive property 0/2 (0%), Fraction/decimal 1/3 (33%), Two-step equations 1/4 (25%). Overall 5/12 (41.7%) — report the cluster breakdown to the client, not just the 41.7%; the single number hides which specific skill is actually broken.

## Score-band tracking table (test-prep example)

SAT Math, baseline 540, 8 sessions over 8 weeks. Target curve set as a range with diminishing returns, not a straight line — and labeled to the client as a heuristic estimate, not a promise.

| Checkpoint | Session | Target range (heuristic) | Actual |
|---|---|---|---|
| Baseline | — | 540 | 540 |
| Mid-point check | 4 | 570-600 | 585 |
| Pre-final check | 7 | 590-630 | 610 |
| Official retest | 8+ | 600-650 | TBD |

Gains are steepest early (540→585 in 4 sessions, +45) and flatten later (585→610 in 3 sessions, +25) — this is the expected shape, not a sign tutoring stopped working; report the curve, not a single end number, so a flattening gain doesn't read as failure.

## Spaced-review log (filled example)

| Skill | First taught | Review 1 | Review 1 result | Review 2 | Review 2 result | Status |
|---|---|---|---|---|---|---|
| Distributive property | Session 1 | Session 3 | 2/2 (100%) | Session 5 | 3/3 (100%) | Retained |
| Fraction/decimal ops | Session 2 | Session 3 | 3/3 (100%) | Session 5 | 2/3 (67%) | Needs 1 more cycle |
| Two-step equations | Session 3 | Session 5 | 3/3 (100%) | — | — | On track, recheck session 7 |

A skill without a logged Review 2 result isn't "done" — it's "not yet checked." Don't report a skill as retained on the strength of Review 1 alone.

## Session plan template (filled example, 45-minute session)

| Segment | Minutes | Content |
|---|---|---|
| Spaced review | 5-7 | 2-3 retrieval items on the prior session's target skill, no notes |
| Targeted instruction | 10-15 | Direct teaching or re-teaching of this session's highest-leverage gap |
| Guided practice | 15-20 | Problems on the target skill, difficulty ramping, tutor narrates reasoning on the first 2-3 |
| Exit check | 5 | 2-3 items, same skill, done independently — logged as same-day evidence only, not proof of retention |

## Gap-ranking worksheet (filled example)

Before sequencing sessions, rank identified gaps by how many current-unit problems each one blocks, not by how the client described the problem.

| Gap | Diagnostic score | Current-unit problems it blocks (of 12 homework problems sampled) | Rank |
|---|---|---|---|
| Distributive property | 0/2 (0%) | 9 of 12 | 1 (highest leverage) |
| Fraction/decimal operations | 1/3 (33%) | 5 of 12 | 2 |
| Multi-step word-problem setup | 2/4 (50%) | 3 of 12 | 3 |

Sequence sessions 1, 2, 3 in rank order — not in the order the client listed complaints, and not by "easiest to fix first."
