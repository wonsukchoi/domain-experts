# Red flags — what an experienced CS professor notices instantly

Smell tests with thresholds. Any one can be innocent; two or three together is a pattern worth acting on.

### A core course's DFW rate jumps more than 8 points year over year with the same instructor and format

Usually means: a single structural change (a new textbook edition's problem sets don't match the exam, a prerequisite course got easier, cohort size crossed the TA-ratio threshold) — rarely a genuine, sudden drop in student aptitude.
First question: "What changed between last offering and this one — instructor, prerequisite course content, TA count, or assessment weighting?"
Data to pull: item-level exam breakdown for both offerings, TA-to-student ratio for both semesters, prerequisite-course grade distributions for the incoming cohort.

### MOSS flags more than roughly 10% of submissions on one assignment above 60% similarity

Usually means: the assignment itself has converged to a small solution space (reused for many semesters, or the starter code constrains the approach too tightly) rather than a sudden wave of cheating.
First question: "Does the similarity survive re-running MOSS with the starter/base file excluded from comparison?"
Data to pull: MOSS output with base-file exclusion applied, plus how many prior semesters this exact assignment prompt has been used unmodified.

### One exam question missed by more than 60% of the class while the overall exam mean is normal

Usually means: the question tests a format or framing students never practiced (an assessment-design gap), not that the specific content was untaught — check whether the same concept scores normally on homework/lab.
First question: "Was this exact question type ever presented as a practice or lab problem before the exam, or only introduced at exam time?"
Data to pull: item-level percent-correct, cross-referenced against the homework/lab problem set covering the same concept.

### A single TA's lab sections show a DFW or grade-dispute rate roughly double the department average

Usually means: grading-calibration drift or inconsistent delivery in that TA's sections, not that those particular students are weaker — this is a staffing/training issue before it's a student issue.
First question: "Did this TA attend and participate in the grading-calibration session, and do their graded samples match the norm set there?"
Data to pull: that TA's graded sample set against the calibration set, section-by-section DFW breakdown across all TAs teaching the same course.

### A capstone or team-project member's version-control commit history is near-zero relative to teammates

Usually means: a free-rider situation building toward a peer-evaluation dispute at the end of the term — catching it at week 4 is a different intervention than catching it at week 14.
First question: "Has this student's contribution been checked against the commit log, not just the team's self-reported peer evaluation?"
Data to pull: full commit history with authorship, timestamped against the project's milestone dates.

### Enrollment in an upper-level elective drops for two consecutive offerings

Usually means: either a prerequisite-chain problem (a required earlier course stopped covering something this course assumes) or the material has gone stale relative to what students perceive as current — rarely a simple popularity swing.
First question: "Did the prerequisite course's syllabus or instructor change in the two years before the drop started?"
Data to pull: prerequisite-course syllabi for the relevant years, exit-survey or advising-note comments from students who didn't enroll.

### A course evaluation score drops sharply the same semester a delivery format changes (in-person to online, or instructor change)

Usually means: the format-change confound, not a genuine quality drop — pooling this semester's evaluation with prior semesters to judge the instructor is a comparison-validity error.
First question: "What else changed the same semester besides the thing being evaluated?"
Data to pull: the evaluation trend over at least 3 prior semesters, and a list of every simultaneous change (format, TA staffing, textbook, room).

### A grade dispute where only the disputed rubric category gets re-checked

Usually means: confirmation-bias risk in the review — re-scoring only the flagged category can miss an error elsewhere or look defensive if the student later finds a different mistake.
First question: "Did the re-grade cover every rubric category, or only the one under complaint?"
Data to pull: the full graded rubric, re-scored fresh against the rubric's descriptor language, independent of the original grader if feasible.

### A tenure dossier's external-letter list is proposed entirely by the candidate with no department-added names

Usually means: weakened evidentiary weight — reviewers and the tenure committee discount a letter set with no arm's-length names added independently of the candidate's suggestions.
First question: "Did the department add names to the external-letter list independently of the candidate's proposed list?"
Data to pull: the solicited-letter list with the source of each name (candidate-proposed vs. department-added) marked.

### A research statement in a tenure dossier reads as a list of paper abstracts with no stated forward agenda

Usually means: the throughline connecting the work is missing, which reads to an external reviewer as a candidate who executed projects rather than one who is building an independent research identity.
First question: "Could a reviewer state the candidate's core research question in one sentence after reading only the first page?"
Data to pull: the research statement's opening paragraph and its explicit 5-year agenda section, if one exists.
