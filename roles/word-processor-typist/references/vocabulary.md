# Word Processor / Typist — Vocabulary

### Style (paragraph/character style)
A named, reusable formatting definition (e.g., "Heading 2," "Body Text," "Emphasis") applied to text, distinct from manually formatting individual characters or paragraphs.
**In use:** "Don't bold that manually — apply the Emphasis character style so it updates if the template changes."
**Common misuse:** Treating "it looks the same on screen" as equivalent to using the actual style — manual formatting and a style can produce identical visual output but behave completely differently under a template change or find-and-replace.

### Redline
A document showing tracked additions and deletions visibly marked (traditionally in red), as opposed to a clean final copy.
**In use:** "Send the redline back to the author, not just the clean version — they need to see what changed."
**Common misuse:** Assuming a redline and a "clean" final copy are interchangeable outputs of the same request — an author who asked for tracked changes wants to see the edit history, not just the end result.

### Track changes / revision marks
The software feature that records edits as visible insertions/deletions with author attribution, rather than silently altering the document.
**In use:** "Turn on track changes before you start — the author wants to see every edit, not just the final text."
**Common misuse:** Accepting changes (making them permanent/invisible) before the author has reviewed them, which removes their ability to see what was altered.

### Sign-off
An explicit, attributable confirmation that a specific document version is approved — a comment, an email, an initialed note — as distinct from a filename or a modification date.
**In use:** "v2 is more recent, but v1 has MJ's sign-off — use v1."
**Common misuse:** Treating "most recently saved" or a filename containing the word "FINAL" as equivalent to an actual sign-off.

### Version reconciliation
The process of determining which of multiple draft files is the authoritative one when they conflict.
**In use:** "Before we send this, we need version reconciliation — there are two drafts with different numbers in them."
**Common misuse:** Skipping this step because "there's probably only one real version," when in practice multiple people editing independently is a common source of divergent drafts.

### Transcription ambiguity
A word, name, or number in dictated or handwritten source material that cannot be determined with confidence from the source alone.
**In use:** "That's a transcription ambiguity — flag it rather than guessing which spelling is right."
**Common misuse:** Resolving an ambiguity by picking whichever option "sounds more likely" without checking available context (prior correspondence, a spelled-out reference elsewhere).

### Homophone-plausible error
A transcription error where the wrong word is grammatically and phonetically plausible in context, making it invisible to a grammar-only read-through.
**In use:** "'Their' vs. 'there' is a homophone-plausible error — spellcheck won't catch it, only a meaning-level read will."
**Common misuse:** Relying on automated spelling/grammar checking alone to catch these, since the tools by design don't flag a correctly-spelled, grammatically-valid wrong word.

### Boilerplate
Standard, reusable document language that doesn't change between instances (e.g., a standard confidentiality clause), as distinct from the variable content specific to this document.
**In use:** "The boilerplate paragraphs are unchanged from the template — only the variable fields (names, dates, amounts) need checking."
**Common misuse:** Assuming boilerplate never needs checking at all — a template update elsewhere can leave stale boilerplate that no longer matches current policy or legal language.

### Mail merge / variable field
A placeholder in a template document that gets populated with instance-specific data (a name, an address, an amount) from a data source.
**In use:** "Check the merge fields populated correctly — a bad data-source match can put the wrong name in the salutation."
**Common misuse:** Checking only that merge fields are populated (not blank), without verifying the populated value is actually correct for that record.

### Widow / orphan
A widow is a single line of a paragraph left alone at the top of a new page/column; an orphan is a single line left alone at the bottom. Both are formatting flaws to control, typically via a paragraph-style setting rather than manual page breaks.
**In use:** "Turn on widow/orphan control in the paragraph style instead of manually inserting page breaks — it'll survive future edits."
**Common misuse:** Fixing a widow/orphan with a manual page break, which then produces a new formatting problem the next time the document's content changes length.

### Delivery note / cover memo
A short note accompanying a finished document that states its source, version basis, and any open items — distinct from the document's own content.
**In use:** "Attach a delivery note listing which draft this was produced from and what's still pending confirmation."
**Common misuse:** Omitting the delivery note because "it's obvious" from context — the next person to touch the file often has none of that context.
