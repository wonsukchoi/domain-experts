# Word Processor / Typist — Red Flags

### Multiple draft filenames in circulation with no explicit sign-off on any of them
- **Usually means:** the approval process happened verbally or over a channel (call, in-person) that never got recorded in the file itself; second-most-likely is that a reviewer's sign-off exists but on a different file than the one currently being worked from.
- **First question:** "Which specific file, and what confirms it was approved — a comment, an email, a verbal instruction someone can point to?"
- **Data to pull:** all draft filenames with modified dates, embedded comments, and any linked email/approval thread.

### A dictated number that doesn't match a figure appearing elsewhere in the case/project file
- **Usually means:** the dictator rounded off from memory; second-most-likely is that the source file itself has since been updated and dictation reflects an older figure.
- **First question:** "Is the dictated figure a deliberate simplification, or does the file's figure need to be used exactly?"
- **Data to pull:** the source document containing the more precise figure, and its own version/date.

### A homophone-plausible name or term appears only once in the document, with no earlier spelled-out reference
- **Usually means:** genuine transcription ambiguity from audio; second-most-likely is a first-time-mentioned name that was never given clearly.
- **First question:** "Has this name appeared anywhere else — prior correspondence, a contact list, an earlier document?"
- **Data to pull:** correspondence history or contact records for both plausible spellings.

### Manual formatting overrides appear scattered through an otherwise templated document
- **Usually means:** a prior editor worked around a style limitation instead of fixing the style itself; second-most-likely is content pasted in from an external source that carried its own formatting.
- **First question:** "Is this override intentional (a genuine exception) or did it happen because the right style wasn't available/known?"
- **Data to pull:** the document's style-template definition, to check whether an appropriate style already exists.

### A tracked-changes document has unresolved comments left over from a prior review round
- **Usually means:** the prior round wasn't fully closed out before the document moved forward; second-most-likely is the comments were addressed verbally but never marked resolved.
- **First question:** "Were these comments actually addressed, or does this need to go back to the original reviewer?"
- **Data to pull:** the comment thread itself and any related communication about resolution.

### Dictation quality is poor enough that multiple consecutive words are unclear
- **Usually means:** recording equipment or environment issue, not a one-off unclear word; second-most-likely is the dictator was speaking too fast or with significant background noise.
- **First question:** "Is a clearer recording available, or does this section need the dictator to re-record?"
- **Data to pull:** the raw audio file's metadata (recording device, environment) if available.

### A "FINAL" or "APPROVED" label appears in a filename but the document has no actual sign-off content
- **Usually means:** someone labeled it optimistically or prematurely, not maliciously; second-most-likely is the approval happened but wasn't documented in the file.
- **First question:** "Who applied this label, and can they confirm what approval it refers to?"
- **Data to pull:** the file's edit history/metadata (who last modified it and when) as a starting point for tracing the label's origin.

### A style guide's stated rule conflicts with an established pattern already used earlier in the same document
- **Usually means:** the document was worked on by multiple people applying different references; second-most-likely is the style guide was updated after the earlier sections were written.
- **First question:** "Which convention does the author actually want going forward — the guide's literal rule, or consistency with what's already in the document?"
- **Data to pull:** the specific style-guide section in question and the earlier instances of the pattern in the document.
