# Retail/Wholesale Buyer — Playbook

## Open-to-buy calculation (filled example)

| Component | Value |
|---|---|
| Planned sales | $150,000 |
| Planned markdowns | $20,000 |
| Planned end-of-month (EOM) inventory | $180,000 |
| **Subtotal (needs)** | $350,000 |
| Beginning-of-month (BOM) inventory on hand | $160,000 |
| Inventory currently on order | $40,000 |
| **Subtotal (already committed)** | $200,000 |
| **Open-to-buy (retail)** | $350,000 − $200,000 = **$150,000** |
| Planned initial markup (IMU) | 55% |
| **Open-to-buy (cost)** | $150,000 × (1 − 0.55) = **$67,500** |

**Use:** Recalculate this table regularly against actual sales and inventory data, not just at the start of the season — a buy placed against a stale OTB figure risks over- or under-investing relative to real current demand.

## Sell-through checkpoint and markdown decision (filled example)

| Checkpoint | Planned sell-through | Actual sell-through | Gap | Action |
|---|---|---|---|---|
| 4-week | 35% | 22% | −13 points | **Early markdown triggered** |

**Markdown calculation:** 800 remaining units at $60 retail (cost $27, 55% IMU) → 20% markdown → new retail = $60 × 0.80 = **$48**.

**Use:** Act at the first checkpoint showing a meaningful gap — waiting for a later checkpoint or end-of-season data converts a manageable early markdown into a deeper, more costly one.

## GMROI calculation (filled example)

| Component | Value |
|---|---|
| Cost of goods for this buy | $67,500 |
| Net sales achieved (after markdowns) | $130,000 |
| Gross margin $ | $130,000 − $67,500 = $62,500 |
| Average inventory cost held during the season | $33,750 |
| **GMROI** | $62,500 ÷ $33,750 ≈ **1.85** |
| Margin % (for comparison) | $62,500 ÷ $130,000 = 48.1% |
| Department GMROI benchmark | 2.5 |
| **Result** | 1.85 < 2.5 → **underperforming on capital efficiency despite a solid-looking 48.1% margin** |

**Use:** Always calculate GMROI alongside margin percentage — a healthy margin percentage can still hide weak capital efficiency if inventory turned slowly or required markdown activity to clear.

## Vendor terms negotiation checklist

1. Unit cost — confirm against comparable vendor benchmarks.
2. Cash discount percentage — negotiate alongside unit cost, not as an afterthought.
3. Dating terms — align payment timing with expected sell-through period where possible.
4. Compliance requirements — confirm chargeback terms for late shipment, mislabeling, or packaging non-compliance, and commit to enforcing them.

## Assortment classification table

| Classification | Management approach |
|---|---|
| Core/replenishable | Ongoing sell-through monitoring, reorder as needed, minimal markdown assumption |
| Fashion/trend | Defined markdown cadence at set checkpoints, no reorder assumption once initial buy is placed |

**Use:** Confirm each item's classification before setting its reorder or markdown plan — misapplying either approach to the wrong item type either overstocks (reordering fashion goods) or unnecessarily discounts (markdown-cadencing core basics).

## Buy performance memo — filled example

> **Season Buy Performance Review — Women's Outerwear, Fall**
> Open-to-buy: $150,000 retail / $67,500 cost (55% IMU).
> 4-week sell-through: 22% actual vs. 35% planned — early 20% markdown taken on remaining units.
> **Realized margin: 48.1% (looks strong on its own). Realized GMROI: 1.85, below the department's 2.5 benchmark** — indicating weaker capital efficiency than the margin percentage alone suggests, driven by the sell-through shortfall.
> **Recommendation: Investigate assortment/pricing for this SKU group ahead of next season; apply earlier markdown triggers at the 4-week checkpoint rather than waiting for later data.**
