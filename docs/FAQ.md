## Why did lint fail?

Run `python3 scripts/lint_roles.py <slug>` and read the actual error — common causes:

- `spec: 2` missing from frontmatter (new roles must be spec 2, no exceptions).
- Missing one of the `references/` trio (`red-flags.md`, `vocabulary.md`, `playbook.md`/`artifacts.md`).
- Banned filler phrase present ("leverage", "utilize", "responsible for", unqualified "stakeholders", etc — full list in [AUTHORING.md](../AUTHORING.md)).
- An idea stated twice (dedup rule) — restating a principle as both a heuristic and a framework step.
- A `references/` link in "Going deeper" that doesn't resolve.

## Why is my new role missing from ROADMAP.md / the README counts?

`generate_roadmap.py` only counts **git-tracked** roles. Run `git add roles/<slug>` before `python3 scripts/generate_roadmap.py` — order matters (see [AGENTS.md § Rules](../AGENTS.md#rules)).

## Why did my push get rejected / conflict?

Another session may be committing to `main` concurrently — this repo has no locking. Always `git pull --rebase --autostash` before pushing. If you hit a real conflict, it's almost always two roles added in parallel — resolve by keeping both.

## Can I hand-edit ROADMAP.md, the README role-count block, or data/roles.json?

No. All three are generated by `scripts/generate_roadmap.py`. Edit the roles, then regenerate.

## Do I need to be a real practitioner to add a role?

No — set `metadata.maturity: draft`. If you are one, or had one review it, set `reviewed-by-practitioner` and say so in the PR (no verification required, good faith). See [CONTRIBUTING.md § Practitioner review](../CONTRIBUTING.md#practitioner-review).

## What if a role is regulated (law/medicine/finance/tax/safety)?

Add the disclaimer blockquote right after the H1 — copy the pattern from `roles/lawyer-contracts/SKILL.md`. Lint checks for its presence on flagged categories.

## Who runs `npm publish`?

The maintainer only — it needs interactive 2FA, so an agent can't do it. Agents can bump `package.json` version and tag; publish itself is a manual step (see [AGENTS.md § Release](../AGENTS.md#release-npm)).

## Where do parity/eval questions come from and how do I add more?

`evals/parity/harvest_stackexchange.py` harvests real practitioner-answered questions. See [docs/CONTEXT.md](./CONTEXT.md) for current parity-set coverage per role — thin sets are a known gap, growing them is higher priority than adding new roles right now.

## I found something wrong in a role I actually work in — what's the fastest fix?

File a [practitioner-correction issue](../../../issues/new/choose) — no PR skills needed, this is the single most valuable contribution type.
